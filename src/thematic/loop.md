# Игровой цикл

## Назначение

Раунд длится 3 часа. Игрок получает роль в живом поселении, закрывает ее повседневные обязанности, добывает ресурсы, поддерживает других жителей и реагирует на угрозу, которая усиливается по ходу смены. Раунд заканчивается общим итогом поселения и отдельными итогами персонажей.

## Циклы игры

| Масштаб | Действия игрока | Результат |
| :--- | :--- | :--- |
| 10–60 секунд | Осмотреть обстановку, выбрать инструмент, выполнить действие, сообщить о результате | Локальное изменение мира и новая информация |
| 2–10 минут | Закрыть потребность, изготовить предмет, оказать помощь, выполнить поручение | Ресурс, услуга, долг или социальное последствие |
| 10–30 минут | Организовать производственную цепочку, расследовать угрозу, подготовить оборону | Изменение готовности поселения к следующей фазе |
| 3 часа | Пережить эскалацию и довести личную или групповую цель до финала | Итог раунда без переноса силы в следующую смену |
| Между раундами | Просмотреть хронику, открыть справочные и косметические элементы | Знание и самовыражение, но не преимущество в силе |

## Базовая петля персонажа

1. Оценить состояние тела, рабочее место, приказы и ближайшие угрозы.
2. Выбрать приоритет: выживание, обязанность роли, помощь группе или личная цель.
3. Получить недостающий ресурс через производство, обмен, просьбу, кражу или насилие.
4. Выполнить действие и создать наблюдаемое последствие для других игроков.
5. Передать информацию, изменить план и подготовиться к следующему уровню эскалации.

Голод и жажда создают регулярный спрос на еду и воду, но не заменяют ролевые задачи. Обычный персонаж, начавший раунд без дефицита, не обязан заниматься питанием в первые минуты и получает время встроить его в рабочий цикл.

## Взаимозависимость ролей

Ни одна роль не должна полностью обеспечивать себя едой, лечением, экипировкой и информацией одновременно. Основная производственная цепочка:

`добытчик → сырье → ремесленник → инструмент или оружие → защитник → безопасность производства → пища и вода → работники`

Разрыв цепочки должен иметь несколько игровых решений: замена специалиста, ручной труд с меньшей эффективностью, обмен с другой группой, разбор запасов или рискованная экспедиция. Потеря одной роли не должна автоматически завершать раунд.

## Эскалация цикла

| Время | Давление на игрока | Ожидаемое решение |
| :--- | :--- | :--- |
| 0:00–0:20 | Низкое; локальные бытовые проблемы | Освоить роль, найти коллег, проверить запасы |
| 0:20–1:10 | Умеренное; первые признаки основной угрозы | Наладить производство, собрать сведения, выбрать приоритеты |
| 1:10–2:20 | Высокое; дефицит, травмы и конфликт интересов | Координировать поселение, жертвовать второстепенными целями |
| 2:20–2:50 | Предельное; основная угроза становится необратимой | Выполнить условие спасения, победы или эвакуации |
| 2:50–3:00 | Итоговое; новые длинные задачи не выдаются | Зафиксировать последствия и завершить личные сцены |

## Правила результатов

- Общий итог определяется состоянием поселения и основной угрозы, а не числом убийств само по себе.
- Личный итог показывает судьбу персонажа и выполнение его целей отдельно от общего результата.
- Досрочный финал допустим только при необратимом общем исходе по правилам `modes/endings.md`.
- Смерть одного персонажа не завершает раунд и не отменяет действия остальных.
- Награды между раундами не повышают характеристики, стартовые ресурсы или доступ к более сильным ролям.

## Edge cases

| Ситуация | Правило |
| :--- | :--- |
| Сервер стартует с малым числом игроков | Критические обязанности объединяются или переводятся в упрощенный режим; таймлайн не сокращается автоматически |
| Игрок подключается после старта | Получает доступную роль, краткую сводку текущей фазы и обычное для этой роли снаряжение; пропущенное время не компенсируется ресурсами |
| Ключевой специалист погиб или вышел | Его функцию можно выполнить менее эффективно альтернативным способом; исключение допускается только для заранее объявленного условия финала |
| Основная угроза устранена слишком рано | Запускается досрочный успешный эпилог только при выполнении всех обязательных условий; иначе раунд продолжается с последствиями |
| Поселение фактически уничтожено | Система проверяет необратимость исхода и запускает досрочный провальный эпилог, не ожидая таймера |

## Критерии приемки

- Полный раунд от старта до завершения занимает ровно 180 минут без досрочного финала.
- Каждая фаза меняет доступные события или интенсивность угроз и сообщает игрокам о переходе через игровые признаки.
- В каждой критической производственной цепочке есть хотя бы один запасной путь после потери роли.
- Итоговый экран разделяет общий результат, судьбы персонажей и личные цели.
- Ни одна постоянная награда не изменяет боевые, производственные или выживательные параметры персонажа.
