# Алхимия

## 1. Роль системы

Алхимия закрывает нишу медицины, ядов, утилитарных зелий и торговли редкими реагентами. Это не "магия в бутылке", а ремесло: игрок ищет сырье, понимает свойства ингредиентов, подбирает сосуд, температуру и порядок смешивания.

В раунде алхимик должен быть полезен без прямого боя: лечить, готовить противоядия, снабжать экспедиции, продавать редкие смеси, подделывать простые напитки, делать опасные вещества для криминальных сюжетов.

## 2. Игровой цикл

1. Найти сырье: травы, грибы, минералы, органы существ, кристаллы, воду, спирт, кровь, слизь.
2. Подготовить сырье: измельчить, высушить, перегнать, выжать сок, очистить от примесей.
3. Собрать рецепт: основа + активный компонент + стабилизатор + необязательная добавка.
4. Провести процесс: смешивание, нагрев, охлаждение, настаивание или перегонка.
5. Получить продукт: зелье, яд, мазь, порошок, эссенцию, испорченную смесь или опасную реакцию.

## 3. Практика в игре

| Ситуация | Что делает игрок | Что должно произойти |
| :--- | :--- | :--- |
| Нужно быстро лечить раны | Смешивает лечебную траву, воду и стабилизатор в колбе | Получает слабое зелье лечения с понятным эффектом и малым риском |
| Нужен яд | Берет токсичный гриб, спирт, концентрирует через перегонку | Получает яд, который можно добавить в еду, напиток или шприц |
| Ресурсов мало | Заменяет редкий компонент похожим по свойству | Рецепт работает хуже или дает побочный эффект |
| Игрок ошибся с температурой | Перегрел смесь | Часть реагентов сгорает, появляется дым, токсин или бесполезная жижа |
| Алхимик хочет заработать | Варит стабильные зелья партиями | Торговец/заказчик принимает товар по качеству и объему |

## 4. Оборудование

| Предмет | Роль | Зависимости |
| :--- | :--- | :--- |
| Ступка и пестик | Измельчение трав, минералов, костей | Предметы, звук работы, время действия |
| Колба | Малые рецепты и эксперименты | Контейнер реагентов, нагрев, осмотр содержимого |
| Котел | Массовое приготовление простых смесей | Температура, топливо, объем, доступ к воде |
| Перегонный аппарат | Очистка, спирты, эссенции, концентраты | Машинерия, трубы/пары, взрыв при ошибке |
| Сушилка/пресс | Подготовка растений и органов | Ботаника, кулинария, хранение |
| Алхимический стол | Рабочее место для сложных рецептов | UI рецепта, навыки, доступ профессии |

## 5. Ингредиенты и свойства

Ингредиент не обязан быть жестко привязан к одному рецепту. Лучше хранить у него набор свойств, чтобы рецепты собирались из категорий.

| Тип свойства | Пример | Для чего нужно |
| :--- | :--- | :--- |
| Лечение | плоть, кровь, ожог, холод, яд | Медицинские зелья и мази |
| Вред | яд, кислота, жар, мороз, удушье | Яды, ловушки, опасные смеси |
| Тело | кровь, голод, жажда, сон, боль | Выживание и медицина |
| Движение | ускорение, слабость, судороги | Баффы, дебаффы, контроль |
| Мана | восстановление, истощение, проводимость | Магия и кристаллы |
| Стабильность | стабилизатор, катализатор, примесь | Качество результата и риск реакции |

## 6. Рецепт

Минимальная структура рецепта:

| Поле | Описание |
| :--- | :--- |
| Основа | Вода, спирт, масло, кровь, слизь или другой носитель |
| Активный компонент | То, ради чего варится смесь |
| Стабилизатор | Снижает шанс порчи и побочных эффектов |
| Процесс | Смешать, нагреть, настоять, перегнать, охладить |
| Качество | Сила эффекта, длительность, вкус/запах, цена |
| Ошибка | Что выходит при неправильном компоненте или процессе |

Рецепты должны быть читаемыми для контента: дизайнеру нужно добавлять зелья без переписывания кода. Система должна поддерживать как точные рецепты, так и рецепты по тегам свойств.

### 6.1. Единицы и сосуды

Все количества реагентов задаются в условных единицах `u`.

| Сосуд | Вместимость | Назначение |
| :--- | ---: | :--- |
| Колба | 50 u | Опыт или одна малая порция. |
| Бутылка | 100 u | Хранение и продажа готового продукта. |
| Котел | 500 u | Партия простого продукта. |
| Перегонный аппарат | 250 u | Очистка и концентрация одной смеси. |

Переливание сохраняет состав и качество смеси. Если свободного объема не хватает, остаток остается
в исходном сосуде. Смешивание не может создать объем сверх суммы входных реагентов, кроме рецепта,
который явно задает газообразный или твердый побочный продукт.

### 6.2. Выбор реакции

1. После изменения состава система собирает все рецепты, для которых хватает реагентов и выполнены
   условия сосуда, температуры и процесса.
2. Точный рецепт имеет приоритет над рецептом по тегам.
3. Среди рецептов одного типа выбирается рецепт с наибольшим числом обязательных условий.
4. Если специфичность одинакова, реакция не начинается, а проверка контента сообщает о конфликте
   prototype ID. Порядок загрузки YAML не влияет на результат.
5. За один шаг расходуется одна порция рецепта. После этого состав проверяется снова.

### 6.3. Этапы процесса

| Процесс | Условие старта | Стартовое время | Прерывание |
| :--- | :--- | ---: | :--- |
| Смешивание | Сосуд открыт, реагенты добавлены | Мгновенно | Не прерывается. |
| Нагрев | Сосуд на источнике тепла | 10 с после достижения порога | Снятие с нагрева ставит процесс на паузу. |
| Настаивание | Сосуд закрыт | 60 с | Открытие сбрасывает прогресс. |
| Перегонка | Смесь в аппарате, приемник установлен | 30 с | Потеря нагрева или приемника ставит процесс на паузу. |
| Охлаждение | Температура ниже порога рецепта | 10 с | Повторный нагрев ставит процесс на паузу. |

Стартовые температурные зоны: холодная — ниже `10 °C`, комнатная — `10–39 °C`, горячая —
`40–79 °C`, кипение — `80–109 °C`, перегрев — от `110 °C`. Конкретный рецепт может сузить
диапазон. Температура должна оставаться внутри него все время этапа.

## 7. Качество и ошибки

Качество хранится целым числом от `0` до `100`.

`Quality = clamp(round(IngredientQuality × 0.5 + Process × 0.3 + Skill × 0.2 - Contamination), 0, 100)`

- `IngredientQuality` — средневзвешенное качество израсходованных ингредиентов.
- `Process` — `100`, если температура и время выдержаны; за каждые 5 секунд вне диапазона
  снимается 10 пунктов.
- `Skill` — прогресс навыка алхимии от 0 до 100.
- `Contamination` — сумма долей посторонних реагентов: 1 пункт за каждый процент примеси,
  максимум 50.

Качество `0–24` дает испорченный продукт, `25–49` — слабый, `50–79` — обычный,
`80–100` — качественный. Качество изменяет силу и длительность эффекта в диапазоне
`0.5–1.25` от базового значения, но не меняет сам тип эффекта.

| Причина | Последствие |
| :--- | :--- |
| Не тот компонент | Слабый эффект, побочка или пустая смесь |
| Грязный сосуд | Примесь попадает в результат |
| Перегрев | Испарение, дым, пожар, взрыв для летучих веществ |
| Недостаток стабилизатора | Эффект сильный, но короткий или опасный |
| Низкий навык | Игрок видит меньше подсказок и чаще портит сложные рецепты |

Ошибки должны давать игровую сцену, а не просто удалять ресурсы. Дым, запах, изменение цвета, ожог рук, испачканный стол и подозрительная бутылка дают другим игрокам повод реагировать.

## 8. Эффекты продуктов

| Категория | Примеры |
| :--- | :--- |
| Медицина | лечение ран, ожогов, яда, кровопотери, усталости |
| Выживание | сытость, жажда, согревание, сон, бодрость |
| Магия | восстановление маны, усиление проводимости, временная защита |
| Криминал | яды, наркотики, паралич, ослепление, усыпление |
| Утилити | свет, дым, клей, растворитель, очистка, приманка |
| Экономика | дорогие эссенции, партии зелий, заказы фракций |

## 9. Информация для игрока

Алхимик не должен играть в слепую таблицу. У системы должны быть способы узнать свойства:

1. Осмотр растения или минерала дает грубое описание.
2. Книга/рецепт показывает известную комбинацию.
3. Навык алхимии раскрывает больше свойств.
4. Пробование опасно, но может дать прямое знание.
5. Торговцы и учителя могут продавать редкие рецепты.

## 10. Зависимости для реализации

| Система | Что нужно |
| :--- | :--- |
| Реагенты | Хранение объема, смешивание, метаболизм, перенос в еду/напитки |
| Контейнеры | Колбы, бутылки, котлы, загрязнение, подписи, осмотр |
| Температура | Нагрев, охлаждение, пороги реакции |
| Ботаника | Растения как источник свойств и редких компонентов |
| Медицина | Типы урона, лечение, передозировка, побочные эффекты |
| Магия | Мана, эссенции, кристаллы, взаимодействие с заклинаниями |
| Экономика | Цена, качество, заказы, контрабанда |
| Логи | Кто сварил, что добавил, кто применил продукт |

## 11. MVP

1. Реагентные контейнеры: колба, бутылка, котел.
2. 10-15 базовых ингредиентов с тегами свойств.
3. 8-10 рецептов: лечение, противоядие, яд, сон, бодрость, мана, дым, испорченная смесь.
4. Простые процессы: смешать, нагреть, перегнать.
5. Качество результата и ошибки.
6. Осмотр смеси: цвет, запах, примерная опасность для обученного персонажа.

## 12. Что не делать в первой версии

Не делать сотни рецептов до готового каркаса. Не делать алхимию полной заменой медицины. Не давать мгновенные сильные баффы без цены, запаха, риска или следов. Не прятать всю механику за UI: игроки должны видеть бутылки, дым, котел, подписи и последствия в мире.

## 13. Пограничные случаи

| Ситуация | Результат |
| :--- | :--- |
| Сосуд удален во время процесса | Процесс прекращается; существующий состав не исчезает. |
| Два игрока работают с одним сосудом | Сервер применяет завершенные действия последовательно; UI обновляется после каждого. |
| Смесь подходит двум равным рецептам | Реакция не запускается, конфликт фиксируется проверкой контента. |
| Источник тепла пропал | Нагрев и перегонка ставятся на паузу, температура естественно падает. |
| Грязный сосуд пуст | Загрязнение остается до промывки и влияет на следующую смесь. |
| Объем результата больше приемника | Заполняется доступный объем; остаток остается в аппарате. |

## 14. Ожидаемое поведение

1. Точный рецепт всегда выбирается вместо подходящего тегового рецепта.
2. Порядок добавления реагентов влияет только на рецепты, которые явно требуют порядок.
3. Процесс нагрева не продвигается до достижения температурного диапазона и ставится на паузу при
   выходе из него.
4. Чистая смесь из ингредиентов качества 100 при правильном процессе и навыке 100 получает
   качество 100.
5. Посторонние реагенты уменьшают качество и не исчезают без указанной реакции.
6. Переливание между сосудами сохраняет состав, объем и качество без дублирования реагентов.
7. Удаление сосуда, пользователя или источника нагрева не создает продукт и не удаляет реагенты
   молча.
8. Логи содержат пользователя, сосуд, израсходованные реагенты, prototype ID реакции и результат.
