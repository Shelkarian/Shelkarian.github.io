# Структура раунда

## Основные параметры

| Параметр | Значение |
| :--- | :--- |
| Плановая длительность | 180 минут |
| Число фаз | 5 |
| Начало отсчета | После завершения загрузки карты и выдачи ролей |
| Завершение отсчета | В момент перехода к экрану итогов |
| Допустимое досрочное завершение | Только через подтвержденное условие финала |
| Продление | Не предусмотрено; административная пауза останавливает таймер |

## Фазы

| Фаза | Время раунда | Длительность | Состояние мира | Назначение |
| :--- | :--- | :--- | :--- | :--- |
| Экспозиция | 0:00–0:20 | 20 минут | Относительно спокойно | Вход в роль, проверка запасов, первые контакты |
| Завязка | 0:20–1:10 | 50 минут | Появляются признаки угрозы | Производство, расследование, формирование конфликтов |
| Эскалация | 1:10–2:20 | 70 минут | Угроза действует открыто | Дефицит, травмы, нарушение инфраструктуры, борьба за решения |
| Кульминация | 2:20–2:50 | 30 минут | Основное условие исхода активно | Последняя попытка спасти поселение или завершить план фракции |
| Эпилог | 2:50–3:00 | 10 минут | Исход зафиксирован | Личные сцены, подсчет результатов, завершение раунда |

## Переходы между фазами

Переход выполняется один раз при достижении временной границы. Он не откатывается при изменении состояния мира. Каждая фаза наследует последствия предыдущей: разрушения, запасы, тела, договоренности и активные эффекты не сбрасываются.

| Переход | Обязательное изменение |
| :--- | :--- |
| Экспозиция → Завязка | Игрокам становится доступен хотя бы один проверяемый признак основной угрозы |
| Завязка → Эскалация | Основная угроза получает возможность напрямую нарушать жизнь поселения |
| Эскалация → Кульминация | Активируется конечная цель или необратимый процесс с ясным способом повлиять на исход |
| Кульминация → Эпилог | Система фиксирует общий исход и прекращает запуск новых крупных событий |
| Эпилог → Итоги | Управление персонажами прекращается, выводится отчет раунда |

Переход обозначается игровыми средствами: изменением освещения или окружения, звуком, сообщением доступной связи либо наблюдаемым событием. Служебное название фазы можно показывать в интерфейсе, но оно не заменяет события в мире.

## Эскалация

Интенсивность задает бюджет одновременного давления, а не обязательный список событий.

| Фаза | Активные очаги угрозы | Повреждение критической инфраструктуры | Новый глобальный кризис |
| :--- | :--- | :--- | :--- |
| Экспозиция | 0–1 локальный | Не допускается случайным событием | Нет |
| Завязка | 1–2 локальных | Временно и обратимо | Не более 1 |
| Эскалация | 2–4 | Допускается при наличии способов ремонта или обхода | Не более 2 одновременно |
| Кульминация | До 5 либо 1 глобальный | Допускается как часть условия финала | Только связанный с основным исходом |
| Эпилог | Новые не создаются | Сохраняется текущее состояние | Нет |

Случайное событие не должно одновременно лишать поселение воды, пищи, лечения и безопасной зоны без заранее телеграфированного способа предотвратить или обратить результат.

## Паузы и сбои

- Административная пауза останавливает фазовый таймер, метаболизм, урон по времени и таймеры событий.
- Краткая потеря соединения отдельным игроком не останавливает раунд.
- После восстановления сервера фаза и ее оставшееся время восстанавливаются из состояния раунда; пропущенные события не запускаются пачкой.
- Если обязательное событие перехода не может появиться, фаза меняется по времени, а оператор получает диагностическое уведомление. Раунд не зависает.

## Edge cases

| Ситуация | Правило |
| :--- | :--- |
| Условие финала выполнено до кульминации | Проверяется досрочный финал; при неполном условии мир сохраняет результат, но идет дальше по таймлайну |
| Условие финала не разрешено к 2:50 | Выбирается итог по состоянию мира и запускается эпилог |
| Все живые игроки стали призраками | Проверяется досрочное поражение; при наличии контролируемых живых персонажей раунд продолжается |
| Антагонисты отсутствуют | Эскалацию обеспечивают мировые угрозы; фазы не пропускаются |
| Игроки устранили все активные угрозы | До кульминации сохраняется период восстановления; новые события подчиняются бюджету текущей фазы |

## Критерии приемки

- Границы фаз наступают на 20, 70, 140, 170 и 180 минутах активного времени.
- Административная пауза не расходует время раунда и временные показатели персонажей.
- Ни один штатный путь не продлевает раунд свыше 180 минут.
- При каждом переходе создается хотя бы один воспринимаемый игроками сигнал.
- После сбоя таймер продолжает текущую фазу, не повторяя уже завершенные переходы.
- В эпилоге не запускаются новые глобальные кризисы и задания, рассчитанные более чем на 10 минут.
