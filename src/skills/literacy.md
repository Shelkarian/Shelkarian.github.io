# Грамотность

Грамотность определяет, может ли персонаж работать с письменным текстом: читать объявления, писать
записки, понимать учебники, оформлять приказы, пользоваться печатями и хранить информацию между
персонажами.

Это не замена языкам. Язык отвечает за смысл речи, грамотность — за доступ к письменной форме.
Персонаж может говорить на человеческом общем, но не уметь читать приказ на этом же языке.

## 1. Зачем механика нужна

В средневековом сеттинге письмо должно быть не бесплатным UI для всех, а отдельным социальным
инструментом. Тогда писцы, управленцы, ученые, жрецы, торговцы и дворяне получают практическую роль,
а документы становятся игровыми объектами.

Грамотность нужна для:

- приказов, долговых расписок, договоров и судебных решений;
- книг навыков и обучения;
- карт, заметок, табличек, объявлений и тайных записок;
- различия между устной договоренностью и письменным доказательством;
- зависимости неграмотных персонажей от тех, кто умеет читать и писать.

## 2. Игровой цикл

### Этап 1: персонаж получает доступ к тексту

Игрок находит бумагу, книгу, объявление, табличку или документ. Игра проверяет не “видит ли он UI”,
а может ли персонаж разобрать содержимое.

| Источник | Что проверяется | Результат |
| :--- | :--- | :--- |
| Бумага или приказ | Грамотность на языке документа | Читает текст или видит непонятную письменность |
| Учебник навыка | Грамотность и требования книги | Может учиться или только держит предмет |
| Записка другого игрока | Грамотность на языке записки | Получает информацию или вынужден искать читателя |
| Печать на документе | Способность осмотреть документ | Может понять знак власти даже без полного чтения |

### Этап 2: персонаж применяет текст

После чтения игрок решает, что делать с информацией: передать устно, переписать, скрыть, подделать,
использовать как доказательство или обменять.

Важный дизайн-момент: грамотность должна создавать действия вокруг текста, а не просто блокировать
кнопку. Неграмотный персонаж не “выключен”, он вынужден обращаться к другим игрокам.

### Этап 3: текст меняет ситуацию

Документ становится причиной действия: стража получает приказ, торговец требует оплату, крестьянин
несет расписку, жрец читает запретный фрагмент, ученик пытается понять учебник.

Так письмо входит в обычную SS14-логику: не катсцена и не квестовый журнал, а предмет в мире,
который можно потерять, украсть, испортить, спрятать, показать или подделать.

## 3. Чтение

Чтение должно проверяться в момент открытия письменного объекта.

| Состояние персонажа | Что видит игрок |
| :--- | :--- |
| Грамотен на языке текста | Обычный текст |
| Знает устный язык, но неграмотен | Непонятные буквы, каракули или частично читаемые фрагменты |
| Не знает язык, но грамотен в другой письменности | Чужую письменность без смысла |
| Не знает язык и неграмотен | Набор знаков без названия языка |
| Администратор или лог | Исходный текст и технические данные |

Для MVP лучше использовать бинарную проверку: грамотен / неграмотен. Плавное раскрытие текста по
проценту навыка можно оставить для книг обучения или будущей системы частичного знания.

## 4. Письмо

Письмо должно проверяться до сохранения текста на предмет.

| Автор | Итог |
| :--- | :--- |
| Грамотен на выбранном языке | Текст записывается нормально |
| Не грамотен на выбранном языке | Запись запрещается или сохраняется как каракули |
| Не знает язык | Нельзя выбрать язык для письма |
| Пишет под диктовку | Возможна отдельная механика: грамотный пишет, неграмотный диктует |

Для практики лучше не давать неграмотному игроку писать нормальный текст “ради удобства”. Иначе
механика сразу теряет смысл: договоры, приказы и учебники снова становятся обычным OOC-текстом.

## 5. Связь с языками

Грамотность должна храниться отдельно от знания устного языка.

| Устный язык | Письменность | Пример |
| :--- | :--- | :--- |
| Знает | Знает | Дворянин говорит и пишет на человеческом общем |
| Знает | Не знает | Крестьянин понимает речь, но не читает приказ |
| Не знает | Знает | Ученый умеет читать древние знаки, но не говорит на языке свободно |
| Не знает | Не знает | Чужой текст остается непонятным |

Для текущей базы языков:

| Язык | Письменность в MVP |
| :--- | :--- |
| Человеческий общий | Обычная грамотность |
| Дворфийский | Вариант человеческого общего, но может иметь отдельный стиль текста |
| Эльфийский | Отдельная грамотность, если язык реализуется как отдельный |
| Гоблинский | Отдельная грамотность, если гоблины получают письменность |

Если дворфийский остается вариантом человеческого общего, его не надо превращать в отдельный барьер
для чтения. Достаточно названия, стиля и культурного использования.

## 6. Обучение

Текущая система навыков уже поддерживает обучение через предмет: книга запускает do-after и добавляет
прогресс навыка повторяющимися порциями. Для грамотности это можно использовать, но сам навык
грамотности пока нужно добавить отдельно.

| Способ | Как должен работать |
| :--- | :--- |
| Стартовая роль | Писцы, дворяне, жрецы, ученые и управленцы получают грамотность на старте |
| Учебник | Чтение учебника добавляет прогресс грамотности |
| Учитель | Игрок с грамотностью ускоряет обучение другого игрока |
| Практика | Частое чтение простых текстов может давать малый прогресс |
| Магия или ритуал | Временное понимание текста без постоянного навыка |

MVP можно ограничить стартовой выдачей и одной учебной книгой. Учитель, диктовка и магическое
понимание — хорошие расширения после того, как заработают базовые проверки чтения и письма.

## 7. Роли

| Роль | Стартовая грамотность | Зачем |
| :--- | :--- | :--- |
| Лорд | Да | Приказы, законы, вассальные договоры, публичные решения |
| Рыцарь | Частично | Может читать приказы, грамоты и военные распоряжения, но не обязан вести учет |
| Управляющий городом | Да | Административные распоряжения, списки работ, городские объявления |
| Казначей | Да | Расписки, учет, налоги, торговые документы, долговые записи |
| Кузнец | По роли | Мастер должен читать заказы и схемы; подмастерье может зависеть от наставника |
| Плотник | По роли | Чертежи, заказы на постройки, ведомости материалов |
| Каменщик | По роли | Планы стен, складские записи, строительные поручения |
| Инженер | Да | Схемы механизмов, инструкции, расчеты, чертежи |
| Корчмарь | Частично | Заказы, долги, объявления, учет запасов; может не быть полноценным писцом |
| Травник | Да | Рецепты, записи о травах, лечебные заметки, предупреждения о ядах |
| Горожанин | Обычно нет | Базовый житель не обязан читать; грамотность зависит от происхождения |
| Торговец | Да | Долги, контракты, цены, учет товара, письма поставщикам |
| Стражник | Частично | Должен узнавать печати, простые приказы и розыскные объявления |
| Староста | Частично | Деревенские списки, сборы, простые распоряжения, связь с городом |
| Фермер | Обычно нет | Создает зависимость от старосты, торговца или писца при договорах и налогах |
| Крестьянин | Нет | Не читает документы сам и вынужден доверять посредникам |
| Переселенец | Нет | Новая низовая роль без доступа к письменной власти и обучению на старте |

Грамотность не должна быть просто “баффом умных ролей”. Ее задача — создавать социальную зависимость:
кто-то пишет, кто-то подтверждает, кто-то несет документ, кто-то не может проверить, что ему подсунули.

## 8. Предметы

| Предмет | Роль в системе |
| :--- | :--- |
| Бумага | Носитель короткого текста |
| Перо | Инструмент письма |
| Печать | Подтверждение происхождения или решения |
| Книга навыка | Предмет обучения |
| Табличка / вывеска | Публичная информация в мире |
| Договор / расписка | Социальное обязательство |
| Запретная книга | Рискованный источник знания |

Бумага и перо уже есть в прототипах Respiral. Проверка грамотности при использовании бумаги пока не
найдена.

## 9. Требования к системе

1. Письменный объект хранит исходный текст и язык/письменность.
2. При чтении игра проверяет грамотность читателя.
3. При письме игра проверяет грамотность автора.
4. Неграмотный не получает исходный текст через UI.
5. Админские логи и инструменты разбора сохраняют оригинал.
6. Печати и визуальные признаки документа можно распознавать отдельно от полного чтения.
7. Учебники навыков могут требовать грамотность, если это не ломает ранний доступ к ремеслам.
8. Устная речь и письменность не смешиваются в один флаг.

## 11. Ожидаемое поведение

| Проверка | Результат |
| :--- | :--- |
| Грамотный открывает бумагу на известном письменном языке | Видит исходный текст |
| Неграмотный открывает бумагу на родном устном языке | Видит непонятную письменность, а не исходный текст |
| Неграмотный пытается написать приказ | Запись не создается или создается как каракули |
| Персонаж без знания языка пытается писать на нем | Не может выбрать этот язык |
| Игрок читает книгу навыка без грамотности | Не получает прогресс, если книга требует чтения |
| Игрок читает книгу навыка с грамотностью | Запускается обучение через do-after и прогресс |
| Админ разбирает конфликт вокруг записки | Видит оригинальный текст, автора и технические данные |
